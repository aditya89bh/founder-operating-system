# Founder Operating System v1.0.0

The first public release. Founder OS is a local-first, deterministic operating
system for founders: six independent engines and a thin integration layer,
driven entirely by the `founder-os` CLI. No AI, no network services, no
dashboards.

## Highlights

- **Six engines.** Memory, decisions, priorities, goals, projects, and reviews,
  each backed by SQLite behind a small `Store` protocol.
- **Integration layer.** The operating loop (`FounderSnapshot` + health
  indicators), historical insights (growth across reviews), and a combined
  founder report exported to Markdown and JSON.
- **One-command demo.** `founder-os demo` loads a realistic dataset into
  temporary databases and prints the status, insights, and report. It is safe to
  run any time and writes nothing to your real databases.
- **Examples package.** A deterministic demo dataset and five founder scenarios,
  with an end-to-end walkthrough and a checked-in sample report.

## Installation

```bash
git clone https://github.com/aditya89bh/founder-operating-system.git
cd founder-operating-system
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

Requires Python 3.11 or newer.

## Try it

```bash
founder-os demo
founder-os --help
```

## Compatibility and scope

This release is intentionally local-first and deterministic. There is no hosted
backend, account system, or telemetry. Each engine stores its data in a local
SQLite database under `~/.founder-os/` by default; paths are configurable per
command via `--db` options.
